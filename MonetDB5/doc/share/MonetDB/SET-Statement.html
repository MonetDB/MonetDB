3.4.2 DECLARE Statement

The user can declare local variables to hold scalar values. 

     DECLARE myvar integer;
     SET myvar=21;

This area requires further extension to make it more usefull.

3.4.3 SET Statement

MonetDB/SQL comes with a limited variable scheme. Globel, session based variables can be set using the construct: 

     SET <variable>=<string>
     SET <variable>=<boolean>
     SET <variable>=<int>

Their type is inherited from the literal value supplied. The SQL engine comes with a limited set of environment variables to control its behavior.

  • The debug variable takes an integer and sets the server global debug flag. (See MonetDB documentation) It also activates the debugger when the query is being executed.
  • The explain variable takes a string value and when set to 'plan' shows the MAL plan before it is being executed. When set to 'trace' it produces the execution trace information for each query. When set to 'profile', it keep track of the SQL global performance in a table called profile.

By default all remaining variables are stored as strings and any type analysis is up to the user. The can be freely used by the SQL programmer for inclusion in his queries. [TODO, a little more flexibilty would mean a lot]
